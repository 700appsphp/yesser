Assumptions:

  • API is JSON API
  • Each module will have one listview to list items and one show view, can be retrieved by object id.
  • modules on system are:
  • Organization about
  • Agencies
  • Agencies categories
  • E-Services categories
  • E-services
  • RSS url
  • Complain (Email or Amer agency ID)
  • FAQ
  • Polls
  • Poll have submit answer view to submit answers
  • and results view to view results on one poll
  • Changes are recorded in one module, only for log, and mobile has access to it.
  • Changes are recorded in that format (Module name, object id, Action, timedate)
  • actions are: Create, update, delete only.
  • list view can have ‘created_after’ argument to show last date mobile retrieved data from server
  • ‘created_after’ format is  yyyy-mm-dd_hh:mm:ss (example: ‘2014-04-30_12:05:00’)
  • if no ‘created_after’ argument included, server will assume that it’s the first time and include all data in model
  • Log module are keeping last 30 days changes only, and every 35 days remove the last 5 days logs.
  • Mobile should assume if it didn’t update itself within 30 days that it’s the first time to run and retrieve all data again.
